Виртуальная машина и ассемблер для звездной анимации

Описание проекта

Проект представляет собой полноценную виртуальную машину с собственным ассемблером, процессором и стековой архитектурой. Реализована анимация взмахов крыльев птицы и вывод текста "МАТАН" на псевдографическом экране 40×40.

Архитектура
stars.asm (исходный код)
    ↓ ассемблер (asm.cpp)

bite_code.txt (байт-код)
    ↓ процессор (processor.cpp)
    
видеопамять (RAM[0..1599]) → вывод в консоль

Компиляция и запуск

Сборка

bash

chmod +x build.sh

./build.sh

Скрипт создаёт два исполнительных файла:

a.out — ассемблер (транслирует .asm в байт-код)

b.out — процессор (исполняет байт-код)

Запуск

bash

# 1. Ассемблирование

./a.out stars.asm bite_code.txt

# 2. Исполнение

./b.out bite_code.txt

Программа stars.asm
Алгоритм анимации
Инициализация — очистка экрана

10 циклов взмахов крыльев — увеличение/уменьшение "размаха"

Вывод текста "МАТАН" — по пикселям в видеопамять

Повтор анимации — 10 циклов в обратную сторону

Координатная система
Экран 40×40, ячейки видеопамяти:

text
RAM[0]      → (1,1)   левый верхний угол
RAM[39]     → (40,1)  правый верхний
RAM[40]     → (1,2)
...
RAM[1599]   → (40,40) правый нижний
Формула перевода адрес → (x, y) в подпрограмме :2:

python
x = (адрес % 40) + 1
y = 40 - (адрес // 40)
Геометрия крыла
Подпрограмма :5 проверяет, нужно ли рисовать пиксель:

c
if (RCX < (RDX * sqrt((RBX-20)^2)) / 10)
    // Рисовать
Создаётся параболический профиль крыла, коэффициент RDX меняется от 5 до 30 и обратно.

Формат байт-кода
Файл bite_code.txt:

text
<количество_команд> <команда1> <команда2> ... -1
Пример:

text
9 1 0 1 0 42 0 43 0 -1 -1
Видеопамять
Размер: 5000 ячеек (используется 1600 для экрана 40×40)

Символ 32 — пробел (пустой пиксель)

Символ 35 — # (заполненный пиксель)

Символ 42 — * (в подпрограмме :7)

Пример вывода
При запуске b.out на экране отображается:

Анимация машущих крыльев (10 циклов)

Текст МАТАН крупными буквами

Анимация в обратную сторону (10 циклов)
